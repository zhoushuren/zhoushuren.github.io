kubernetes-集群部署文档

工具介绍

Flannel：

Flannel是CoreOS团队针对Kubernetes设计的一个网络规划服务，简单来说，它的功能是让集群中的不同节点主机创建的Docker容器都具有全集群唯一的虚拟IP地址。 用我自己的话理解就是在传输层封装了另一个IP地址。

Docker：

不用多说，容器技术。

etcd

ETCD是用于共享配置和服务发现的分布式，一致性的KV存储系统。

Services

搭建Docker私有仓库。

DevOps流程

本地基于TDD模式开发->提交Git -> CI测试用例 -> CI build Dockerfile 到镜像 -> CI push Docker镜像->CI Update kubernetes(让集群拉取最新Docker镜像)

搭建K8s流程。

  1. 搭建Master服务器。 配置yum源
vim /etc/yum.repos.d/virt7-docker-common-release.repo

[virt7-docker-common-release]
name=virt7-docker-common-release
baseurl=http://cbs.centos.org/repos/virt7-docker-common-release/x86_64/os/
gpgcheck=0

然后安装

yum -y install --enablerepo=virt7-docker-common-release kubernetes etcd flannel

安装的这几个东西以上都解释过。 先配置etcd

vim /etc/etcd/etcd.conf
//生产环境不要用0.0.0.0 换成内网网卡的IP
ETCD_LISTEN_CLIENT_URLS="http://0.0.0.0:2379" 
ETCD_ADVERTISE_CLIENT_URLS="http://0.0.0.0:2379"

启动etcd systemctl start etcd

配置apiserver

 vi /etc/kubernetes/config

KUBE_API_ADDRESS="--insecure-bind-address=0.0.0.0"

# The port on the local server to listen on.
 KUBE_API_PORT="--port=8080"

# Port minions listen on
KUBELET_PORT="--kubelet-port=10250"

# Comma separated list of nodes in the etcd cluster
KUBE_ETCD_SERVERS="--etcd-servers=http://pink-master:2379"

# Address range to use for services
KUBE_SERVICE_ADDRESSES="--service-cluster-ip-range=10.254.0.0/16"

# default admission control policies
#KUBE_ADMISSION_CONTROL="--admission-control=NamespaceLifecycle,NamespaceExists,LimitRanger,SecurityContextDeny,ServiceAccount,ResourceQuota"

# Add your own!
KUBE_API_ARGS=""

配置flanneld 这里ETCD监听2379端口，并且PREFIX为/pink/network

 vim  /etc/sysconfig/flanneld
 
# etcd url location.  Point this to the server where etcd runs
FLANNEL_ETCD_ENDPOINTS="http://pink-master:2379"
# etcd config key.  This is the configuration key that flannel queries
# For address range assignment
FLANNEL_ETCD_PREFIX="/pink/network"

# Any additional options that you want to pass
#FLANNEL_OPTIONS=""

解释一下下面的配置，第一行建立和flanneld一样的命名空间。并且配置一段JSON。Network为Node所在的子网。

etcdctl mkdir /pink/network
etcdctl mk /pink/network/config "{ \"Network\": \"10.0.0.0/16\", \"SubnetLen\": 24, \"Backend\": { \"Type\": \"vxlan\" } }"

用一个shell脚本重启一下他们,并且设置为开机子启动，顺便看看状态。

for SERVICES in etcd kube-apiserver kube-controller-manager kube-scheduler flanneld; do
     systemctl restart $SERVICES;
     systemctl enable $SERVICES;
     systemctl status $SERVICES;
      done

如不能正常启动请看日志 journalctl -xe

装Node节点

yum -y install flannel kubernetes

设置 kubectl

enabled=1
gpgcheck=0
repo_gpgcheck=0
gpgkey=http://mirrors.aliyun.com/kubernetes/yum/doc/yum-key.gpg 
http://mirrors.aliyun.com/kubernetes/yum/doc/rpm-package-key.gpg 
EOF
